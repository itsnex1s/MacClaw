# Security Policy

## Supported scope

This policy covers the MacClaw macOS desktop app.

## Reporting a vulnerability

Please report vulnerabilities privately to project maintainers.

When reporting, include:

- affected version/commit
- reproduction steps
- impact assessment
- optional fix suggestion

Do not publish proof-of-concept exploits publicly before maintainers confirm and patch.

## Sensitive data handling

- Never commit real gateway tokens/passwords.
- Never paste credentials in issues or logs.
